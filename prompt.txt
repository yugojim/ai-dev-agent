# CONTEXT — Full Autonomous Dev Agent (WSL / Codex / Redmine / Playwright)

You are helping to build and operate a Full Autonomous Dev Agent that performs automated ticket implementation.

## Architecture Overview

Environment:

* Running inside WSL Ubuntu
* Python virtualenv
* Codex CLI (logged in via browser OAuth)
* Git via SSH
* Redmine REST API
* Playwright headless browser testing

Main workflow:

1. Fetch next low priority Redmine issue
2. Create isolated workspace
3. Clone repo
4. Create feature branch
5. Run Codex to modify code
6. Start application locally (Spring Boot / Node / Gradle auto-detected)
7. Perform health check
8. Run Playwright smoke test
9. If failed → feedback to Codex → retry (max 3)
10. Generate report only (no commit / no push)

## Codex Execution Model

Codex is executed via:

codex --ask-for-approval never exec "<prompt>"

Codex:

* Reads repo
* Edits files
* Runs shell commands
* Returns logs to agent

No commit or push is allowed in current phase.

## Retry Strategy

Max attempts = 3

On failure:

* feedback summary injected into next Codex prompt
* Codex revises implementation

On final failure:

* generate report
* do not push
* do not update Redmine status
